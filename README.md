# MLB Pitcher Projections with XGBoost

Projecting next-season ERA and WHIP for MLB pitchers using gradient boosted trees trained on Statcast-era data (2015–2025). Built as a portfolio project exploring whether Statcast pitch-level features (velocity, movement, barrel rate, Stuff+) improve projections beyond what traditional stats offer, benchmarked against established systems like Steamer and ZiPS.

Accompanying blog post: *[link TBD]*

## Getting started

Requires Python 3.10+.

```bash
git clone https://github.com/djfeldman94/pitcher-projections.git
cd pitcher-projections

python -m venv .venv
source .venv/bin/activate

pip install -r requirements.txt

jupyter notebook --notebook-dir=notebooks
```

## Notebooks

Run these in order — each builds on the output of the previous one.

| Notebook | What it does |
|----------|-------------|
| `01_data_exploration` | Pulls pitching data via `pybaseball`, explores the 390+ available columns, and gets familiar with the shape of the dataset |
| `02_feature_selection` | Selects ~60 features from the raw data, builds lag columns for multi-year trends, and engineers derived features like velocity slopes and luck-gap indicators |
| `03_modeling` | Trains XGBoost regressors to predict next-season ERA and WHIP from the engineered feature matrix |
| `04_evaluation` | Compares model accuracy against Steamer, ZiPS, and naive baselines (prior year stats) using MAE and residual analysis |
